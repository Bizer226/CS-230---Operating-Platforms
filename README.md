# CS-230---Operating-Platforms

1. Briefly summarize The Gaming Room client and their software requirements. Who was the client? What type of software did they want you to design?

The Gaming Room is a company that has developed a game for Android called Draw It or Lose It, where teams compete to guess what is being drawn.  The images are rendered by the application from an image library consisting of 200 images.  The Gaming Room tasked my company with developing a web-based version of the game that can be played on multiple platforms, including Windows, Linux, MAC, and iOS.

2. What did you do particularly well in developing this documentation?

I thought that my research of the different web development platforms was thorough and helped give me an understanding of how web-based development works.

3. What about the process of working through a design document did you find helpful when developing the code?

The process of working through a design document was a tremendous help in giving me a solid foundation for developing the code.  It helped me outline the functions the code would need and gave me a path to follow when developing the code.

4. If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?

If I could choose one part of the design document to revise, it would have to be the domain model.  I think I got a little lost in the specifics of the domain model and would have benefited from a second pass where I made the domain model more clear and concise for the client to understand it.

5. How did you interpret the user’s needs and implement them into your software design? Why is it so important to consider the user’s needs when designing?

Reading through the prompts for each project helped me understand the user's needs to implement them into my software design.  Understanding that The Gaming Room's highest priority was to remain within their budget helped me choose a development platform that would suit their needs.  It is essential to consider the user's needs when designing because their needs are the endgame for the code.  If a program doesn't satisfy the user's needs, then it's not the right program.

6. How did you approach designing software? What techniques or strategies would you use in the future to analyze and design a similar software application?

I approached designing software one step at a time.  First, understanding the client's needs in order to choose the proper platform and creating an outline of the application.  Using the Software Design Document as a roadmap for developing the application will be something I do often in the future.  It helps create a solid foundation for the eventual code development.
